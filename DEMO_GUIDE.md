# Mercury Demo 骨架 - 功能说明

## 界面布局

Mercury 采用经典的三栏布局设计：

### 1. 顶部标题栏
- 显示应用名称 "Mercury"
- 支持窗口拖拽（-webkit-app-region: drag）

### 2. 左侧 Feed 侧边栏（280px）
**功能区域：**
- **添加订阅按钮**：蓝色主按钮，点击弹出"添加订阅功能（占位）"提示
- **刷新按钮**：白色次要按钮，点击弹出"刷新功能（占位）"提示

**标签筛选区：**
- 显示所有可用标签：全部、技术、设计、产品
- 点击标签可切换选中状态（蓝色高亮）
- 当前默认选中"全部"

**Feed 列表：**
- 显示 3 个 Mock 订阅源：
  - Hacker News (12 篇未读)
  - 阮一峰的网络日志 (5 篇未读)
  - 少数派 (8 篇未读)
- 点击 Feed 可切换选中状态（浅蓝色背景）
- 显示未读数量徽章

### 3. 中间文章列表（380px）
**头部控制区：**
- 标题："文章列表"
- 筛选按钮：全部（默认选中）、未读、已读

**文章列表：**
- 显示当前选中 Feed 的文章
- 每篇文章显示：
  - 标题
  - 作者和发布时间
  - 摘要（最多 2 行）
  - 标签（如果有）
  - 未读状态（蓝色圆点）
- 点击文章可查看详情（浅蓝色背景 + 左侧蓝色边框）

**Mock 文章数据：**
1. "Show HN: I built a RSS reader with AI features" - 未读，带标签
2. "The Future of Web Development" - 已读
3. "Understanding TypeScript Generics" - 未读

### 4. 右侧阅读区（flex: 1）
**文章头部：**
- 文章标题（24px 大字）
- 元信息：作者、发布时间、查看原文链接
- 操作按钮行：
  - 📝 AI 摘要（点击弹出占位提示）
  - 🌐 AI 翻译（点击弹出占位提示）
  - 🏷️ 添加标签（点击弹出占位提示）
  - 📤 导出（点击弹出占位提示）

**文章内容区：**
- AI 摘要区域（灰色背景，蓝色左边框）- 当前为空
- AI 翻译区域（灰色背景，蓝色左边框）- 当前为空
- 文章正文（HTML 渲染）：
  - Introduction 段落
  - Key Features 列表
  - Technical Stack 说明

**空状态：**
- 未选中文章时显示：📖 图标 + "选择一篇文章开始阅读"

## 交互流程

### 基础浏览流程
1. 启动应用 → 默认选中第一个 Feed (Hacker News)
2. 中间列表显示该 Feed 的 3 篇文章
3. 默认选中第一篇文章
4. 右侧显示文章详情

### Feed 切换流程
1. 点击左侧其他 Feed
2. 中间列表更新为该 Feed 的文章
3. 右侧清空（显示空状态）

### 文章阅读流程
1. 点击中间列表的任意文章
2. 文章高亮显示（浅蓝色背景 + 左侧蓝色边框）
3. 右侧显示文章完整内容

### 标签筛选流程
1. 点击左侧标签
2. 标签高亮显示（蓝色背景）
3. 当前仅 UI 交互，未实现真实筛选逻辑

### AI 功能流程（占位）
1. 点击"AI 摘要"按钮 → 弹出 alert("AI 摘要功能（占位）")
2. 点击"AI 翻译"按钮 → 弹出 alert("AI 翻译功能（占位）")
3. 点击"添加标签"按钮 → 弹出 alert("添加标签功能（占位）")
4. 点击"导出"按钮 → 弹出 alert("导出 Markdown 功能（占位）")

## 样式特点

### 配色方案
- 主色：#409eff（蓝色）
- 背景色：#f5f7fa（浅灰）
- 文字色：#2c3e50（深灰）
- 边框色：#e4e7ed（浅灰）
- 次要文字：#909399（中灰）

### 交互反馈
- 按钮 hover：边框变蓝 + 文字变蓝
- 列表项 hover：背景变浅灰
- 选中状态：浅蓝色背景
- 未读文章：更浅的灰色背景

### 滚动条样式
- 宽度：8px
- 轨道：#f1f1f1
- 滑块：#c1c1c1
- 滑块 hover：#a8a8a8

## 技术实现

### 组件结构
\`\`\`
App.vue
├── TitleBar.vue
└── main-content
    ├── FeedSidebar.vue
    ├── ArticleList.vue
    └── ReaderView.vue
\`\`\`

### 数据流
- App.vue 维护所有状态（feeds, articles, selectedFeedId, selectedArticleId）
- 通过 props 向下传递数据
- 通过 emits 向上传递事件
- 使用 computed 计算派生状态

### Mock 数据
- 3 个 Feed
- 4 个 Tag
- 3 篇 Article
- 1 篇完整 ArticleContent

所有数据都硬编码在 App.vue 中，后续将替换为真实的数据库查询和 API 调用。
