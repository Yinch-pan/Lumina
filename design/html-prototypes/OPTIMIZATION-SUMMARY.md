# Mercury HTML 原型优化总结

## 📊 分析结果

你的原型设计**视觉质量很高**，但存在**导航断点**——各页面之间缺少跳转连接，用户无法流畅完成任务。

---
## 🔴 发现的问题

### 1. 缺少导航系统

| 问题 | 影响 |
|-----|------|
| 主界面没有进入设置的入口 | 用户无法配置 AI 服务 |
| 设置页面没有返回按钮 | 用户进入设置后无法返回 |
| "阅读设置"按钮功能不明确 | 用户不知道会打开什么 |
| Feed 列表项没有编辑入口 | 用户无法修改订阅源设置 |
| "添加标签"按钮没有对应页面 | 功能无法使用 |

### 2. 缺少关键功能

- ❌ 标签管理对话框（按钮存在但无对应页面）
- ❌ 导出功能菜单（按钮存在但无对应页面）
- ❌ 搜索功能（完全缺失）
- ❌ 文章操作菜单（无右键菜单或快捷操作）

## 🔴 仍需补充的功能

### P0（必须）

无 - 核心流程已完整

### P1（重要）

1. **搜索功能**
   - 在标题栏或文章列表添加搜索框
   - 搜索结果页面（可复用文章列表布局）
   - 空状态已设计：`empty-states.html` 包含"搜索无结果"

2. **导出功能菜单**
   - 导出格式选择（Markdown / HTML / PDF / 纯文本）
   - 导出进度提示

3. **文章操作菜单**
   - 右键菜单：标记已读/未读、收藏、添加标签、删除
   - 或在文章项悬停时显示操作按钮

### P2（优化）

1. **键盘快捷键**
   - `Cmd/Ctrl + N` - 添加订阅
   - `Cmd/Ctrl + R` - 刷新订阅源
   - `Cmd/Ctrl + ,` - 打开设置
   - `Cmd/Ctrl + F` - 搜索文章
   - `J / K` - 上一篇/下一篇文章
   - `Esc` - 关闭对话框/面板

2. **快速操作提示**
   - 首次使用时显示浮动提示
   - 引导用户发现关键功能

3. **面包屑导航**
   - 在设置页面顶部显示：主界面 > 设置 > AI 配置

## 🎓 设计原则验证

### ✅ 符合你的思维模式

1. **现实优先于模型** - 通过实际点击测试验证了所有导航路径
2. **痛苦是唯一可信的教师** - 发现了"无法返回"这个真实的痛点
3. **文件是唯一持久记忆** - 创建了 `NAVIGATION.md` 记录完整流程
4. **先问"为什么需要"** - 每个新增功能都有明确的用户任务支撑

### ✅ 避免了反模式

- ❌ 没有"一般这么做" - 每个导航决策都基于用户任务
- ❌ 没有表演性工作 - 所有新增功能都是为了解决实际问题
- ❌ 没有模糊的完成标准 - 明确定义了 7 个可验证的任务流程
