# 模块 B 用户使用说明：清爽阅读与正文清洗

## 1. 功能说明

模块 B 会在用户打开文章时自动处理原网页内容：

- 提取正文。
- 去掉导航、广告、评论、脚本等干扰内容。
- 保留段落、标题、图片、代码块、表格、链接。
- 生成适合 AI 摘要、翻译和 Markdown 导出的正文。
- 在右侧 ReaderView 中显示清爽阅读界面。

用户不需要手动点击“清洗”。只要打开文章，系统会自动完成。

## 2. 基本使用流程

1. 在左侧选择一个订阅源。
2. 在中间文章列表点击一篇文章。
3. 右侧阅读区会显示“文章加载中”。
4. 加载完成后显示清洗后的正文。
5. 如果文章已有标签，标题下方会显示标签。
6. 可以点击顶部按钮执行摘要、翻译、添加标签、标记未读、导出。

如果正文清洗失败，阅读区会显示提示，并提供“查看原文”链接。

## 3. 阅读设置

进入设置页后，可以调整：

| 设置项 | 作用 |
|---|---|
| 字体大小 | 调整正文阅读字号 |
| 行距 | 调整长文章阅读密度 |
| 主题 | 在浅色和深色阅读主题之间切换 |

保存阅读设置后，ReaderView 会自动使用新的阅读样式。

## 4. 正文显示规则

ReaderView 优先展示清洗后的 `cleanedHtml`。

正常情况下会保留：

- 标题和正文段落。
- 有序/无序列表。
- 图片和图注。
- 代码块。
- 表格。
- 引用。
- 原文中的安全链接。

会主动移除：

- 广告、侧边栏、评论区、社交分享区。
- 脚本、表单、iframe。
- 不安全链接，例如 `javascript:`。
- 1x1 跟踪像素。

## 5. 常见问题

### 右侧一直没有正文

可能原因：

- 原订阅源只提供摘要，没有完整 HTML。
- 原网页禁止抓取或网络访问失败。
- 原网页结构特殊，正文提取失败。

处理方式：

1. 点击“查看原文”打开源网页。
2. 刷新订阅源后重新打开文章。
3. 如果是固定网站一直失败，把网站 URL 和文章 URL 记录给开发同学，用于补站点规则。

### 图片没有显示

可能原因：

- 源站图片需要登录或防盗链。
- 原图片链接已经失效。
- 图片使用了项目暂未覆盖的懒加载属性。

处理方式：

- 先点击“查看原文”确认源站图片是否正常。
- 如果源站正常而 Mercury 不显示，记录文章 URL 给模块 B 维护者。

### 导出的 Markdown 内容不完整

Markdown 导出依赖 `cleanedMarkdown`。如果打开文章后正文本身不完整，导出也会不完整。

处理方式：

1. 先在 ReaderView 确认正文是否完整。
2. 若 ReaderView 不完整，优先定位清洗规则。
3. 若 ReaderView 完整但 Markdown 不完整，定位导出模块。

### 阅读设置保存后没有变化

可能原因：

- 当前运行环境不是 Electron，无法访问本地设置 API。
- 保存设置失败。

处理方式：

- 在正式 Electron 应用中测试。
- 确认设置页保存时没有错误弹窗。
- 关闭设置页后重新打开文章。

## 6. 演示建议

推荐演示顺序：

1. 添加一个稳定 RSS 源。
2. 打开文章，展示右侧自动出现清爽正文。
3. 指出正文中保留了标题、段落、图片、代码或链接。
4. 如果文章有标签，展示标题下方的标签区域。
5. 打开设置页，调整字号或深色主题，保存后回到阅读区。
6. 点击“查看原文”，对比源网页和清洗后的阅读视图。
7. 点击“导出”，说明导出使用的是模块 B 生成的 `cleanedMarkdown`。

演示时建议使用结构简单、网络稳定的订阅源，避免选择登录墙、强反爬或图片防盗链站点。
