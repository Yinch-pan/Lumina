# 模块 D 功能测试清单

## 测试环境
- 应用已启动
- 至少有一个 Feed 和一篇文章

## 1. 标签管理功能测试

### 1.1 添加标签
**操作步骤：**
1. 点击左侧 Feed，选择一个订阅源
2. 点击中间文章列表中的一篇文章
3. 在右侧 Reader View 中点击"添加标签"按钮
4. 输入标签名称，例如："技术"
5. 点击确认

**预期结果：**
- 弹出输入框
- 输入标签名称后，标签添加成功
- 文章标签列表更新，显示新添加的标签
- 不应有错误提示

### 1.2 查看文章标签
**操作步骤：**
1. 选择一篇已添加标签的文章
2. 查看 Reader View 中的标签显示

**预期结果：**
- 标签正确显示在文章详情中
- 标签格式清晰可读

### 1.3 多次添加相同标签
**操作步骤：**
1. 对同一篇文章重复添加相同标签

**预期结果：**
- 标签不重复
- 或者给出友好提示

## 2. Markdown 导出功能测试

### 2.1 导出单篇文章
**操作步骤：**
1. 选择一篇文章（最好有正文内容的）
2. 点击右侧 Reader View 中的"导出"按钮
3. 选择保存路径和文件名
4. 确认导出

**预期结果：**
- 弹出文件保存对话框
- 默认文件名为文章标题（已清理非法字符）
- 导出成功后显示成功提示
- 打开导出的 .md 文件，应包含：
  - 文章标题（一级标题）
  - 元信息：作者、发布时间、原文链接、标签
  - 正文内容（Markdown 格式）
  - AI 摘要（如果有）
  - AI 翻译（如果有）

### 2.2 导出没有正文的文章
**操作步骤：**
1. 选择一篇还未加载正文的文章
2. 点击导出

**预期结果：**
- 能够导出
- 正文部分显示"（无正文内容）"或原始 HTML

### 2.3 取消导出
**操作步骤：**
1. 点击导出按钮
2. 在文件保存对话框中点击"取消"

**预期结果：**
- 无错误提示
- 不创建文件

## 3. 设置管理功能测试

### 3.1 打开设置页面
**操作步骤：**
1. 点击顶部标题栏的"设置"图标/按钮

**预期结果：**
- 设置页面覆盖整个应用窗口
- 显示三个设置区域：
  - LLM 配置
  - 阅读设置
  - 关于 Mercury

### 3.2 配置 LLM
**操作步骤：**
1. 在 LLM 配置区域填入：
   - Base URL: https://api.openai.com/v1
   - API Key: sk-test123456
   - Model: gpt-4
2. 点击"保存 LLM 配置"按钮
3. 关闭设置页面
4. 重新打开设置页面

**预期结果：**
- 保存成功提示
- 重新打开设置页面后，配置值保持不变
- 配置已持久化到 SQLite 数据库

### 3.3 修改阅读设置
**操作步骤：**
1. 修改字体大小为"大 (18px)"
2. 修改行距为"宽松 (2.0)"
3. 修改主题为"深色"
4. 点击"保存阅读设置"按钮
5. 关闭并重新打开设置页面

**预期结果：**
- 保存成功提示
- 重新打开后，设置值保持不变
- 设置已持久化

### 3.4 关闭设置页面
**操作步骤：**
1. 点击右上角的 ✕ 按钮

**预期结果：**
- 设置页面关闭
- 返回主界面

## 4. 完整流程测试

### 4.1 端到端测试
**操作步骤：**
1. 添加一个 RSS 订阅源（如果没有）
2. 刷新 Feed，获取文章
3. 打开一篇文章，查看正文
4. 给文章添加标签"测试标签"
5. 打开设置页面，配置 LLM（如果有真实 API Key 可以测试 AI 功能）
6. 关闭设置页面
7. 导出当前文章为 Markdown

**预期结果：**
- 所有步骤流畅完成
- 无错误提示
- 导出的 Markdown 文件包含标签信息

## 5. 错误处理测试

### 5.1 无效标签名
**操作步骤：**
1. 尝试添加空标签（只输入空格）
2. 尝试添加特殊字符标签

**预期结果：**
- 空标签应被拒绝或自动过滤
- 特殊字符应能正常处理

### 5.2 无文章选中时的操作
**操作步骤：**
1. 不选择任何文章
2. 点击导出按钮（如果可见）

**预期结果：**
- 给出友好提示
- 不崩溃

### 5.3 导出失败处理
**操作步骤：**
1. 尝试导出到无权限的目录（如 C:\Windows\）

**预期结果：**
- 显示错误提示
- 应用不崩溃
## 6. 数据持久化验证

### 6.1 重启应用
**操作步骤：**
1. 完成上述所有测试
2. 关闭应用
3. 重新启动应用
4. 检查：
   - 文章标签是否保留
   - LLM 配置是否保留
   - 阅读设置是否保留

**预期结果：**
- 所有数据正确保留
- 无数据丢失
