# Mercury 近一周更新梳理（2026-06-18 ~ 2026-06-25）

> 用途：作为 PPT 汇报的内容底稿。每个一级标题可对应一页幻灯片，要点已按"可直接转 bullet"的粒度组织。

## 一、一句话总览

本周 Mercury 完成了一轮**功能密集迭代**，核心是两条主线：

1. **核心功能扩展**：落地 9 项新功能 + 1 项数据库地基重构；
2. **阅读器交互体验（Reader UX）增强**，让阅读页更接近成熟产品。

**规模**：37 个文件改动，约 **+4690 / -95** 行；30 余次提交，集中在 6-22 与 6-24 两天。

## 二、本周主线一：核心功能扩展（9 项）

> 在保持现有三栏布局不变的前提下，对照产品现状梳理出一批值得补齐的能力，只在组件内部扩展交互、在主进程扩展能力。

### 地基：数据库版本化迁移 + WAL
- 用 `PRAGMA user_version` 做有序、事务化迁移，替换原先临时的单表迁移逻辑。
- 开启 WAL 模式 + `busy_timeout`，提升并发读写稳定性。
- 为后续加列、加表、加索引提供可持续的升级机制。

### A 档（核心 5 项）
- **A1 全文搜索（FTS5）**：新增 `entries_fts` 虚拟表，触发器自动同步，文章列表顶部加防抖搜索框。
- **A2 逐段翻译 + 双语对照（流式）**：按段落拆分逐段翻译，单段失败指数退避重试；新增**主进程→渲染进程事件推送通道**，边翻边出双语对照。
- **A3 划词高亮 + 笔记**：新增 `highlights` 表，选中正文弹浮动工具栏（多色高亮 + 加笔记），重开文章按"文本+上下文"重新定位着色。
- **A4 摘要分档**：摘要支持 短 / 中 / 长 三档，对应不同 prompt 与 token 预算。
- **A5 API Key 加密存储**：用 Electron `safeStorage` 加密 LLM API Key，启动时自动把历史明文 key 迁移加密。

### B 档（增强 4 项）
- **B1 LLM 用量统计面板**：按模型 / 日期 / agent 类型聚合 `llm_usage`，设置页新增"用量"标签页（轻量表格 + CSS 柱状，不引重型图表库）。
- **B2 文章收藏 / 星标**：`entries` 加 `is_starred`，列表与阅读页加星标按钮，侧栏加"⭐ 收藏"入口。
- **B3 阅读进度记忆**：`entries` 加 `scroll_percent`，滚动防抖保存、重开文章自动恢复位置。
- **B4 AI 建议标签**：新增 `TagSuggestionAgent`，优先复用已有标签给出候选，**用户勾选确认后才写入**（AI 不自动改标签）。

## 三、本周主线二：阅读器 UX 体验增强

围绕"读得更舒服、操作更顺手"做的一组交互改进：

- **快捷阅读设置面板**：字号 / 行高 / 阅读宽度可即时调节。
- **阅读进度条 + 回到顶部按钮**。
- **上一篇 / 下一篇文章导航按钮**。
- **代码块一键复制按钮**。
- **图片点击灯箱（Lightbox）放大查看**。
- **内嵌"原网页"迷你浏览器**：在应用内查看原页面，不再跳系统浏览器。
- **摘要流式实时显示**：摘要生成支持流式分块回调，边生成边展示进度。

## 四、稳定性与安全加固（harden）

本周在新功能之外，做了多处针对性加固，体现工程质量意识：

- **安全**：API Key 解密失败记录日志；`safeStorage` 不可用时跳过冗余迁移。
- **高亮防护**：高亮颜色白名单 + 函数式 replace，规避属性 XSS 与 `$` 替换 bug。
- **搜索健壮性**：标点符号查询返回空（避免 FTS5 语法错误）；删除订阅源时清理 FTS 索引，避免孤儿数据膨胀。
- **阅读器竞态修复**：滚动保存、摘要写入都绑定文章 ID，避免快速切换文章时数据写错对象。
- **暗色模式适配**：用量表格表头/表尾改用主题变量。
- **灯箱与高亮工具栏冲突处理**：已选中文本时不触发图片灯箱。

## 五、技术亮点（适合单独一页强调）

1. **从"请求/响应"到"事件推送"**：新增主进程→渲染进程的事件通道，支撑流式翻译与流式摘要的实时进度。
2. **版本化数据库迁移**：从临时单表迁移升级为可持续的有序迁移体系 + WAL。
3. **安全意识落地**：密钥加密存储 + 历史明文自动迁移；输入/属性层面的 XSS 防护。
4. **AI 以人为本**：标签建议、摘要分档都强调"AI 给建议、用户做决定"，AI 不擅自改数据。
5. **细节竞态防护**：多处用文章 ID 绑定异步写入，规避切换文章时的错配。
